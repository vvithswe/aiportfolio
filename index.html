Projects

🔓 Project 1: AI Glossary for Non-Technical Professionals

WEEK 1 · PUBLIC · ~1 hour

A plain-English reference explaining AI concepts. Demonstrates you can translate technical ideas for business audiences.

Step-by-step

  1. Open Claude (free at claude.ai). For each concept below, ask Claude to explain it, then rewrite in YOUR words.
  2. Concepts: LLM, Generative AI, RAG, Token, Context Window, Hallucination, Agent, Prompt Engineering, Fine-tuning, Embedding
  3. For each: add a "Why it matters at work" sentence with a concrete example
  4. Add a "How these connect" section: LLM is the brain, RAG gives it company memory, Agent lets it take actions
  5. Add real-world examples: Williams Racing (Rovo agents for F1 operations), HarperCollins (4x reduction in manual work)
  6. Save as projects/01-ai-glossary.md in your GitHub repo

🔓 Project 2: Prompt Engineering Library

WEEK 2 · PUBLIC · ~1.5 hours

A documented collection of prompt patterns with before/after examples showing how iteration improves output.

Step-by-step

  1. Pick 5 tasks: summarize a document, find stale work items, write a status update, compare options, explain a process
  2. Write a "naive prompt" for each (how most people would ask). Run it through Claude. Save the output.
  3. Apply techniques: add role, structure, constraints, examples. Run improved prompt. Save output.
  4. Document: what you changed, why the output improved, which technique had the biggest impact
  5. Create a Prompt Pattern Cheat Sheet: Role Assignment, Structured Output, Few-Shot, Chain of Thought, Constraints
  6. Remove any company-specific data. Use generic examples only.
  7. Save as projects/02-prompt-library.md

🔒 Project 3: Rovo Agent Build – Case Study

WEEK 3 · PRIVATE (sanitized summary public) · ~1.5 hours

Build a real Rovo agent at work. Write a sanitized case study about the process for your portfolio.

Step-by-step

  1. Identify one recurring question in your org ("Where's the doc for X?" or "What's the status of Y?")
  2. Create a Rovo agent: name, role description, system instructions, knowledge sources (specific spaces/projects)
  3. Write 5 conversation starters (pre-written questions users can click)
  4. Test with 10 questions. Rate each: Useful / Partially useful / Wrong
  5. Iterate instructions based on failures. Test again. Document improvement.
  6. Private notes: full config, test results, iteration log, final accuracy
  7. Public case study: "I built an AI agent that [does X]. Here's what I learned." No company names or data.
  8. Save public version as projects/03-rovo-agent-case-study.md

🔒 Project 4: AI Capabilities Audit Framework

WEEK 3 · PRIVATE (template public) · ~1.5 hours

Audit every AI feature in your Atlassian instance. Publish the reusable template for others.

Step-by-step

  1. Open Atlassian Intelligence features documentation
  2. Create a table: Feature | Product | Available? | Currently Used? | Potential Use Case | Priority
  3. Go through every product: Jira, Confluence, JSM, Rovo Search, Rovo Chat, Rovo Agents
  4. Check what's enabled in your instance vs what's available in your plan
  5. For unused features, write one sentence on how it could help a specific team
  6. Highlight top 5 opportunities with a paragraph recommendation each
  7. Share the completed audit with your manager (private). Publish the blank template (public).
  8. Save template as projects/04-ai-audit-template.md

🔓 Project 5: AI Concepts Cheat Sheet for Organizations

WEEK 4 · PUBLIC · ~1 hour

A one-pager designed for sharing with colleagues. Shows you can communicate AI at the organizational level.

Step-by-step

  1. Title: "AI at Work – What Every Team Should Know"
  2. Section 1: "What AI can do today" – 5 capabilities with examples
  3. Section 2: "What AI cannot do" – hallucinations, judgment, data quality dependency
  4. Section 3: "How to get started" – 3 simple actions anyone can take
  5. Section 4: "Staying safe" – data handling, privacy, responsible use
  6. Keep to ONE PAGE. No jargon. Write for someone who uses Jira daily but never thinks about AI.
  7. Save as projects/05-ai-cheat-sheet.md

🔓 Project 6: Agentic AI Tool Comparison

WEEK 4 · PUBLIC · ~1 hour

Compare major AI tools in the enterprise landscape. Shows employers you understand the competitive context.

Step-by-step

  1. Cover: Rovo Agents, Claude Code, GitHub Copilot, Microsoft Copilot for M365, Amazon Bedrock
  2. For each: What it is (1 sentence), Who it's for, What it does (3 points), Where it fits
  3. Add "How they connect" – e.g., Bedrock runs Claude, Claude Code is for devs, Rovo is for knowledge workers
  4. Add "What this means for organizations" – why you need a multi-tool AI strategy
  5. Use only public information. Cite sources.
  6. Save as projects/06-agentic-ai-comparison.md
